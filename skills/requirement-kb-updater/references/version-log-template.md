# 版本变更记录模板

知识库最后一章建议使用：

```md
## <n>. 版本变更记录

| 版本 | 日期 | 变更来源 | 变更说明 |
| --- | --- | --- | --- |
| v1.0 | YYYY-MM-DD | 初始来源 | 初版知识库：整理基础需求、后台配置、客户端展示、图片识别摘要、验收标准。 |
| v1.1 | YYYY-MM-DD | 本地 PRD / Word / PDF / Excel / 用户补充 / 禅道变更 / 原型修订 | 简要说明本次新增、修改、删除的关键规则。 |
```

写法要求：

- 版本号按 `v主版本.次版本` 递增；一般小改动递增次版本。
- 日期使用当前会话日期。
- 变更来源写清楚：本地 PRD/Word/PDF/Excel、用户补充、禅道变更、原型修订、测试反馈等。
- 变更说明要具体，避免“优化文档”“更新内容”这种空话。
- 如果本次删除了某个章节或规则，也要写进变更说明。

中文说明：如果变更来自本地文件，优先写清文件名或相对路径，例如 `本地 PRD：xxx需求说明.md`；如果来自目录扫描，写关键文件而不是笼统写“本地文件”。
