# 前端客户端原型规范

## 当前页面优先

- 先查找并打开当前页面截图、旧 HTML 原型、禅道附件或设计稿；没有当前页面资料时先向用户索要。
- 未获得用户明确授权前，不要重新定义一套视觉风格。
- 新增功能应作为当前页面的增量模块出现，沿用原页面的背景、卡片、按钮、弹窗、字号、间距和状态文案。
- 如果只能找到局部截图，基于截图可见部分延展，不要臆造不可见页面的大改版。

- 若需求涉及 PC 和移动端，必须提供设备切换或响应式预览。
- 用控件模拟后台配置，例如标题是否为空、跳转模式、游戏数量、启用状态。
- 原型需要展示关键交互，不只是静态 UI。
- 点击行为要有反馈，例如 toast、状态切换、页面区域变化。
- 视频类需求可用渐变或占位块模拟视频，但声音开关、播放状态、点击跳转要可交互。
- 样式应贴近原需求截图；已有产品风格优先。
